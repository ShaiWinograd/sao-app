# 6. Open Decisions

## Direct-assignment expiration

Version one:

- Assignment remains pending.
- It auto-rejects only if shift becomes full.

Future decision:

- Whether pending assignments expire after a fixed time.

## Team-leader permissions

Future discussion needed for:

- Attendance oversight
- Customer communication
- Worker issue management
- Project-document visibility
- Team-level form ownership

## Approximate monthly payment

Postponed.

Possible future models:

- Completed approved shifts only
- Completed plus assigned future shifts
- Separate actual and forecast totals

## End-of-shift form ownership

Version one:

- Every worker fills an individual form.

Possible future alternatives:

- Team-leader-only form
- Shared shift form
- Worker forms plus team-leader summary

## Browser location limitations

Web cannot guarantee:

- Background checks when closed
- Reliable periodic checks on all browsers
- Push notifications

Mobile should implement full intended behavior.
